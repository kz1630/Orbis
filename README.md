# Orbis - Система управления страховым агентством

## Описание

**Orbis** — это веб-приложение на ASP.NET Core MVC для управления деятельностью страхового агентства. Система позволяет управлять договорами страхования, клиентами, услугами и пользователями с различными ролями доступа.

## Основные функции

- **Управление пользователями** — поддержка трёх ролей (Администратор, Менеджер, Пользователь)
- **Справочник клиентов** — управление физическими и юридическими лицами
- **Справочник услуг** — каталог страховых услуг с категориями и стоимостью
- **Управление договорами** — регистрация, редактирование и отслеживание договоров
- **Поиск и фильтрация** — поиск договоров по номеру, клиенту, услуге, статусу и периоду
- **Отчёты** — формирование отчётов по договорам за выбранный период
- **Проблемные договоры** — автоматическое выявление просроченных и истекающих договоров

## Технологический стек

- **Backend:** ASP.NET Core 10.0 MVC
- **Database:** SQLite
- **ORM:** Entity Framework Core
- **Frontend:** Bootstrap 5, jQuery
- **Валидация:** jQuery Validation
- **Иконки:** Bootstrap Icons

## Установка и запуск

### Требования
- .NET 10.0 SDK или выше
- Visual Studio 2026 (рекомендуется) или VS Code

### Шаги установки

1. Клонируйте репозиторий:
```bash
git clone <repository-url>
cd Orbis
```

2. Откройте проект в Visual Studio:
```bash
cd Orbis
```

3. Восстановите зависимости:
```bash
dotnet restore
```

4. Запустите приложение (F5 в Visual Studio)

## Модели данных

### User
Представляет пользователя системы с ролью доступа.

### Person
Представляет физическое или юридическое лицо (клиента).

### InsuranceService
Представляет услугу страхования с описанием, стоимостью и сроком действия.

### Contract
Представляет договор страхования, связывающий клиента с услугой.

## Авторизация

Система использует сессии для управления авторизацией. При входе создаётся сессия с информацией о пользователе. Все страницы, кроме страницы входа, защищены глобальным фильтром `AuthorizeFilter`.

## База данных

База данных SQLite создаётся автоматически при первом запуске приложения. Инициализация данных происходит через класс `DbInitializer`, который заполняет БД тестовыми данными:
- 3 пользователя с разными ролями
- 15 клиентов (физических и юридических лиц)
- 10 услуг страхования
- 20 договоров в различных статусах

## Использование

### Вход в систему
1. На странице входа введите логин и пароль
2. Нажмите "Войти"

### Управление договорами
1. Перейдите в раздел "Договоры"
2. Используйте фильтры для поиска нужных договоров
3. Нажимайте иконки для редактирования, просмотра или удаления

### Просмотр отчётов
1. Перейдите в раздел "Отчеты" в навигации
2. Выберите период для отчёта
3. Просмотрите статистику и детализацию

### Просмотр проблемных договоров
1. Перейдите в раздел "Отчеты" → "Проблемные договоры"
2. Система автоматически выявит:
   - Просроченные договоры
   - Активные договоры с истекшим сроком
   - Договоры, срок которых истекает в течение 30 дней
